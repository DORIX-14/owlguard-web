Step 1: Mapping the Attack Surface

Crawling ka main maqsad website ke un pages ko dhundna hai jo Google par nahi dikhte. Hum modern tools jaise Katana use karenge.

Why Katana? Ye JavaScript files ko bhi scan karta hai aur hidden parameters nikalta hai.

Step 2: Live Scanning

Apna crawling command run karein. Hum chahte hain ki tool JS files aur status codes dono fetch kare.

katana -u https://target.com -jc -sc

Step 3: Analysis of Endpoints

Output file ko check karein. Dhyan dein in endpoints par:

  • /api/v1/... (API vulnerabilities ho sakti hain).
  • /dev/test/... (Testing environments jo bhul gaye hon).
  • .env / .git (Secret keys leak ho sakti hain).

Ethical Note

Spidering website par bohot saari requests bhejti hai. Isse kabhi kabhi server slow ho sakta hai. Practice ke liye hamesha permissions lein ya lab environment use karein.
