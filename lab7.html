Step 1: Vulnerability Testing

Sabse pehle check karein ki parameter vulnerable hai ya nahi. URL ke end mein ek single quote (') lagayein. Agar website error deti hai (Jaise: SQL Syntax Error), toh samajh lijiye darwaza khula hai.

Logic: Website ' character ko filter nahi kar rahi, isliye wo database query ko tod raha hai.

Step 2: Automated Scanning (sqlmap)

Manual SQLi mushkil ho sakta hai, isliye hum industry standard tool sqlmap ka use karenge. Sabse pehle databases ki list nikalenge.

sqlmap -u "http://target.com/product.php?id=1" --dbs

Step 3: Data Dumping

Jab database name mil jaye (Example: user_db), toh uske andar ki tables aur columns nikal kar data "Dump" karein.

sqlmap -u "URL" -D user_db -T admin_users --dump

Isse aapko usernames aur hashed passwords mil jayenge.

Safety & Ethics

SQL Injection se aap poora database delete bhi kar sakte hain (DROP TABLE). Isliye kabhi bhi kisi live production site par ye try na karein. Sirf test sites jaise testphp.vulnweb.com par practice karein.
